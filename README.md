# Классификация мусора с использованием кастомной CNN на PyTorch

В данном репозитории содержится Jupyter Notebook (`CNN_garbage_classification.ipynb`), в котором реализован пайплайн глубокого обучения для классификации изображений мусора по категориям с использованием библиотеки **PyTorch**. Архитектура сверточной нейросети (CNN) спроектирована с нуля.

## 🚀 Особенности проекта

- **Автоматическая загрузка датасета**: Использование библиотеки `kagglehub` для бесшовного скачивания актуального датасета Garbage Classification.
- **Кастомный пайплайн данных**: Изменение размера изображений ($64 \times 64$), нормализация тензоров и базовая предобработка.
- **Валидация**: Разделение выборки на обучающую и валидационную в соотношении 80/20 с мониторингом `Loss` и `Accuracy` на каждой эпохе.
- **Визуализация метрик**: Построение графиков сходимости функции потерь и роста точности для оценки переобучения.
---

## 📊 Структура датасета

Проект использует [Garbage Classification Dataset](https://www.kaggle.com/datasets/mostafaabla/garbage-classification), содержащий изображения, распределенные по **12 классам**:

- Бумага (Paper), Картон (Cardboard), Пластик (Plastic), Стекло (Glass), Металл (Metal), Мусор (Trash), Одежда (Clothes), Обувь (Shoes), Батарейки (Battery), Биоотходы (Biological), Биоразлагаемый пластик (Oxo-degradable) и Полипропилен (Polypropylene).

---

## 🏗️ Архитектура модели

Кастомная сверточная нейросеть (`MyCNN`) оптимизирована для извлечения пространственных признаков при относительно небольшом количестве параметров:

1. **Экстрактор признаков (Feature Extractor)**:
   - **Layer 1**: `Conv2d` (3 $\rightarrow$ 32 каналов, ядро $3\times3$) + `ReLU` + `MaxPool2d` ($2\times2$)
   - **Layer 2**: `Conv2d` (32 $\rightarrow$ 64 каналов, ядро $3\times3$) + `ReLU` + `MaxPool2d` ($2\times2$)
   - **Layer 3**: `Conv2d` (64 $\rightarrow$ 128 каналов, ядро $3\times3$) + `ReLU` + `MaxPool2d` ($2\times2$)
2. **Классификатор (Classifier)**:
   - `Flatten` $\rightarrow$ `Linear` (128 _ 8 _ 8 $\rightarrow$ 512 выходов) + `ReLU` + `Dropout(0.5)` для регуляризации.
   - `Linear` (512 $\rightarrow$ 12 выходов для финального распределения по классам).

---

## 📈 Обучение и результаты

- **Функция потерь**: `CrossEntropyLoss`
- **Оптимизатор**: `Adam`
- **Количество эпох**: 25
- **Метрики**:
  - **Точность на обучении (Training Accuracy)**: ~83%
  - **Точность на валидации (Validation Accuracy)**: ~79%
